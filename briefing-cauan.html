📋 1. Visão Geral do Projeto
O que é, para quem é, qual o objetivo

O escritório trabalha com acordos internacionais de previdência social — tratados entre o Brasil e outros países que permitem somar tempo de contribuição no exterior com o tempo no Brasil para fins de aposentadoria. O cálculo principal se chama RMI Pro-rata (Renda Mensal Inicial Proporcional).

Foram criadas duas calculadoras para o site www.acordosinternacionais.com, com perfis de acesso diferentes:

calculadora-segurado.html
Versão pública e gratuita, para o segurado (cidadão comum que quer saber se tem direito ao benefício). Linguagem simples, tutorial embutido, CTA para o escritório.
  • Acesso: público — sem login
  • Tutorial de como baixar o CNIS do gov.br
  • Dois modos: upload de PDF ou preenchimento manual
  • CNIS extrai automaticamente: nome, CPF, data de nascimento, salário e tempo
  • Campos de data de nascimento e sexo (necessários para verificar idade mínima)
  • Resultado em linguagem acessível — 4 casos possíveis (ver seção 7)
  • Botão de contato via WhatsApp ao final (CTA — texto varia por caso)
  • Sem campos de cliente / sem impressão
calculadora-advogado.html
Versão profissional, para advogados e despachantes previdenciários que assinam o plano. Terminologia técnica, campos de cliente, laudo imprimível.
  • Acesso: restrito — somente usuários com assinatura ativa
  • Campos: nome, CPF, data de nascimento, sexo, data da análise, responsável
  • CNIS extrai automaticamente: nome, CPF, data de nascimento, salário e tempo
  • Resultado técnico completo: SB, coeficiente, índice pro-rata, fórmula — 4 casos (ver seção 7)
  • Caso de planejamento (sem idade): RMI projetada + análise de oportunidade
  • Botão "Imprimir / PDF" (window.print() com CSS de impressão embutido)
  • Botão "Limpar" para novo cálculo sem recarregar
  • Sem CTA / sem tutorial
🏗️ 2. Arquitetura Técnica
O que você precisa saber antes de integrar
✓ Sem backend nos arquivos HTML. Toda a lógica (leitura de PDF, cálculo, exibição) roda 100% no navegador do usuário. Você não precisa criar nenhuma API para as calculadoras em si.

Única dependência externa (CDN)

Os dois arquivos carregam uma biblioteca JavaScript de leitura de PDF:

// Incluído no <head> de cada arquivo: <script src="https://cdnjs.cloudflare.com/ajax/libs/pdf.js/3.11.174/pdf.min.js"></script> // Configurado no <script> interno: pdfjsLib.GlobalWorkerOptions.workerSrc = 'https://cdnjs.cloudflare.com/.../pdf.worker.min.js';

Se quiser hospedar localmente para não depender do CDN, baixe os dois arquivos (pdf.min.js e pdf.worker.min.js) e atualize os caminhos.

⚙️ 7. Notas Técnicas Adicionais

Sobre o parser de CNIS

O parser lê o texto extraído do PDF (via PDF.js) usando expressões regulares. A função parseadorCNIS() extrai quatro tipos de dado:

  • Nome: regex busca "Nome:" seguido de letras maiúsculas, para antes de "Data", "CPF", "NIT" ou dois espaços — evita capturar campos adjacentes
  • CPF: regex busca "CPF:" seguido do padrão 000.000.000-00, com formatação automática
  • Data de nascimento: regex busca "Nascimento:" seguido de DD/MM/AAAA
  • Vínculos e salários: pares de datas DD/MM/AAAA e valores monetários 1.234,56 — vínculos com duração negativa ou zero são descartados (proteção contra datas invertidas no PDF)
Atenção sobre datas invertidas: o PDF do INSS imprime a data de emissão no topo do documento. O parser antigo capturava esse par invertido (emissão → data antiga) gerando meses negativos que corrompiam o total. A versão atual filtra if (meses > 0), descartando esses pares. Se o INSS mudar o layout, testar com CNIS real e verificar se o total de meses faz sentido.
O parser foi desenvolvido para o layout atual do PDF emitido em meu.inss.gov.br. Se o INSS mudar o layout do extrato, pode ser necessário ajustar as regex em parseadorCNIS(). Sempre testar com um PDF real antes de lançar.

Sobre o cálculo pro-rata — 4 casos

A lógica avalia os casos nesta ordem (a primeira condição verdadeira encerra o cálculo):

  • Caso 1: Tempo Brasil ≥ carência → totalização desnecessária e prejudicial. Mostra RMI integral estimada sem pro-rata.
  • Caso 2: Brasil + exterior < carência → benefício inviável. Mostra déficit em meses.
  • Caso 2B (planejamento): Carência cumprida por totalização, mas idade atual < mínima (62F / 65M) → mostra quanto falta para a idade, RMI projetada e orientação sobre como maximizar o benefício até lá. Só se avalia se data de nascimento e sexo estiverem preenchidos.
  • Caso 3: Totalização válida e idade cumprida → calcula e exibe RMI pro-rata = (tempoBrasil ÷ tempoTotal) × RMI_teórica.
Carências usadas: Aposentadoria por Idade = 180 meses (15 anos) | Pensão por Morte = 18 meses. Estes são os valores correntes pela legislação (Lei 8.213/91). Verificar periodicamente se houve alteração.
Idades mínimas (EC 103/2019): 62 anos (feminino) e 65 anos (homens). Existem regras de transição para quem já contribuía antes da reforma — a calculadora usa as idades definitivas como referência. Para análises de casos em transição, o advogado deve ajustar manualmente.

Compatibilidade

  • Funciona em todos os navegadores modernos (Chrome, Firefox, Edge, Safari)
  • Responsivo para mobile (layout adapta em telas menores de 600px)
  • Não requer instalação, build ou compilação — são arquivos HTML puros
  • A impressão foi testada em Chrome → "Salvar como PDF" e na impressora do sistema
